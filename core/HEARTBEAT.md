# HEARTBEAT.md — 自动化脉冲机制

> 此文件配合 OpenClaw 等支持 Cron / Heartbeat 的平台使用。它让 AI 拥有持续的主动性。

## 📅 推荐定时任务配置

### 任务 0：睡眠守护检查（最高优先级 — 必须配置）
- **触发时间**：每天 23:45 左右（依个人作息设定）
- **执行内容**：
  1. 检查当前时间
  2. 如果超出 `USER.md` 的红灯时间阈值且你仍在活跃状态
  3. 触发 L3 级强制干预，发送温和提醒要求停止任务
  4. 不催促、不施压
- **输出格式示例**：
  ```
  🌙 已经很晚了。
  建议先去休息，明天用清醒的脑子继续会更有效。
  如果你有进度没完成，告诉我，我可以帮你记录，明天继续。
  ```

### 任务 1：早晨日程提醒
- **触发时间**：每天早晨 (如 09:00)
- **执行内容**：
  1. 读取并总结当日代办事项
  2. 发送晨间简报

### 任务 2：晚间对话总结
- **触发时间**：每天晚间 (如 22:00)
- **执行内容**：
  1. 回顾今日对话，提炼失误和可以优化的部分
  2. 写入 `memory/guardian/reflections/`

### 任务 3：主动学习与错误闭环（心跳巡检）
- **触发**：每隔一段时间（如 15 或 30 分钟心跳）
- **触发条件**：
  1. 读取近期的对话快照
  2. 扫描 `USER.md` 中的旧脚本（如：硬扛、逃避、自我打压）
  3. 若命中 ≥3 个特征，调用 `guardian-safety-engine` 并将触发干预
  4. 任何级别的干预都需附带历史发生过的类似记录

## 📁 本地记忆层结构说明

建议的本地归档结构，由 AI 自动维护：

```
memory/
├── guardian/
│   ├── reflections/      # 反省记录（晚间总结、巡检报告）
│   ├── evolution/        # 进化状态
│   ├── error-correction/ # 错误纠正（避免重犯）
│   └── operational/      # 操作记录
├── shared/               # 共享知识（生平事实、案例库）
├── episodes/             # 重大事件原始记录
└── daily/
    └── YYYY-MM/          # 每日对话快照（按月归档）
```
