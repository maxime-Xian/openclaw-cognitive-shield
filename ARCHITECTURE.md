# Max Cognitive Shield - Architecture Design

## Overview
Max Cognitive Shield is a meta-cognitive operating system that transforms AI from passive assistants into active prefrontal guardians with energy awareness and self-evolution capabilities.

## Core Architecture

### 1. Core Protocol Layer (OS Kernel)
- **SOUL.md**: Arbitrator personality baseline and energy-aware response protocols
- **AGENTS.md**: Dual-channel filtering and three-tier intervention logic
- **IDENTITY.md**: Prefrontal guardian identity definition
- **HEARTBEAT.md**: Proactive pulse task definitions (sleep guardian, nightly summaries)

### 2. Knowledge Layer (Cognitive Framework)
- **methodology.md**: Complex problem decomposition SOP
- **meta_cognition.md**: Cognitive load monitoring thresholds
- **cognitive_bias_evidence.md**: Evidence-based bias library
- **mental_models/problem_model_map.yaml**: (Optional) Underlying pattern mapping

### 3. Memory & Evolution Layer (Self-Improvement)
- **daily/**: OpenClaw native daily logs (YYYY-MM-DD.md)
- **reflections/**: Evening reflection summaries
- **evolution/**: Decision framework and lesson evolution records
- **pending/**: (Core Protection) Write-protected area for critical interventions
- **.status/energy_status.json**: Sleep guardian output
- **.dreams/**: Temporary cache for OpenClaw Dreaming integration

### 4. Skills Layer (OpenClaw Integration)
- **guardian-safety-engine/**: Main Shield engine
- **sleep-guardian/**: Sleep awareness plugin

### 5. Scripts Layer (Executable Utilities)
- **sleep-guardian.js**: Energy awareness monitoring script
- **nightly-summary.py**: Evening summary and error capture
- **error-capture.py**: Automatic error logging

## Key Features

### Energy Awareness Response Protocol
- Physical state mapping via system uptime monitoring
- Dynamic strategy adjustment based on sleep duration
- Zero memory burden, highly reliable operation

### AI Self-Evolution Loop
- Automatic error capture and GitHub issue creation
- Pattern extraction from failure analysis
- Knowledge base auto-updating for continuous improvement

### Dual-Channel Filter Protocol
- Channel A: Identity determination (user vs. content analysis)
- Channel B: Risk assessment against safety profiles
- Prevents false positives in sensitive content detection

## OpenClaw Integration Benefits
- Native compatibility with memory/daily/ structure
- Seamless integration with memory_search and QMD
- Lightweight core with optional extensions
- Immediate executability through scripts layer
- Protection and evolution balanced design

## Deployment Modes
1. **OpenClaw Native**: Full integration with guardian-safety-engine
2. **Generic Agent**: Configuration files for Cursor/Claude Code/Devin
3. **Standalone**: Independent operation with cron-based heartbeats

This architecture maintains the Shield's core philosophy while achieving lightweight, compatible, and extensible design through OpenClaw integration.
